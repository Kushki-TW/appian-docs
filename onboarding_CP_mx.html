¿Cómo realizar el onboarding de subcomercios en México Card Present con Terminales Kushki?

Este proceso aplica al onboarding de subcomercios en el modelo Card Present con terminales en México. A diferencia de otros flujos, el subcomercio no se crea en la consola de Kushki ni genera un Branch ID. En su lugar, se debe registrar usuario en la plataforma Billpocket.

Paso a paso del proceso

Paso 1. Registro del subcomercio en Billpocket

Antes de realizar el registro del submerchant en Appian, el subcomercio debe completar su registro en la plataforma de Billpocket, la URL de la plataforma es proporcionada por Kushki durante el onboarding del PSP (URL agrupadora).

Requisitos para el registro:

  • Correo electrónico único
  • Contraseña segura
  • Teléfono móvil (obligatorio para recibir un código mediante  SMS de validación)

Una vez validado el teléfono, el sistema permitirá continuar con la creación del usuario.

Paso 2. Datos requeridos según el tipo de persona

El sistema solicitará distintos datos dependiendo de si el subcomercio se registra como persona física o moral.

Persona natural o física:

  • Nombre de tu negocio
  • Razon social
  • Industria
  • Giro (actividad específica)

Datos del dueño del negocio (Representante legal):

  • Datos del titular:
  • Nombres y apellidos (del representante legal)
  • Sexo
  • Fecha y lugar de nacimiento (para cálculo automático de CURP)

⚠️ Importante:

Para calcular automáticamente el campo CURP (Clave Única de Registro de Población), es fundamental completar los campos de fecha y lugar de nacimiento.

CURP del dueño del negocio:

  • CURP
  • RFC (debe completarse manualmente luego del registro inicial)

Registro de tarjeta de débito o cuenta CLABE:

Aquí deberás ingresar la cuenta de depósito en la cual llegará el ingreso de las transacciones (cuenta bancaria del PSP).

  • Cuenta de depósito:
  • Tarjeta de débito o
  • Clave interbancaria (CLABE)

Paso 3. Acceso al panel de administración (Web App)

Una vez finalizado el registro, se crea el usuario y se habilita el acceso a la Web App de Billpocket, también conocida como panel de administración.

⚠️ Importante:

  • El usuario debe actualizar su domicilio fiscal, ya que este campo es obligatorio para la afiliación.
  • El usuario debe validar su correo electrónico mediante el enlace enviado al email registrado.

Es fundamental actualizar la información en el panel de administración para finalizar el onboarding.

Paso 4. Obtención del ID de usuario y su uso en Appian

El ID de usuario estará disponible en la plataforma Brain (acceso otorgado al PSP durante su proceso de onboarding), específicamente en el campo "ID Usuario".



Paso 5. Registro del ID de usuario en Appian (campo Open Field)

En Appian, en el Submerchant Onboarding Report debe ingresar el ID de usuario obtenido desde Billpocket en el campo llamado: Open Field o bien también se puede ingresar el correo registrado en la web app de billpocket.

⚠️ Importante:

Este campo funciona como opcional, pero en el flujo Card Present con terminales en México, su uso es obligatorio.

Consideraciones adicionales

Para la operativa Card Presente con terminales, el proceso de onboarding continúa como en otros flujos. Sin embargo, hay diferencias clave:

  • No se genera un Branch ID.
  • En la vista Tracking - Overview, el campo Submerchant ID mostrará el ID de usuario registrado en Open Field.
